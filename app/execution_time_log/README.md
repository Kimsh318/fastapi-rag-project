`execution_time_log` 폴더는 로그 파일에서 데이터를 추출하고, 이를 Elasticsearch에 저장하며, 데이터의 시각화를 담당하는 기능을 수행합니다. 이 폴더 내의 파일들은 로그 파일을 모니터링하고, 로그 데이터를 처리하여 성능 분석 및 모니터링을 가능하게 합니다. 아래는 주요 파일의 역할에 대한 설명입니다.

### 주요 특징

- **자동화된 로그 처리**: 로그 파일의 변경을 실시간으로 감지하고, 데이터 추출 및 저장을 시작합니다.
- **로그 데이터 추출 및 저장**: `extract_data_from_log.py`를 실행하여 로그 파일에서 데이터를 추출하고, 이를 Elasticsearch에 저장합니다.
- **(opt)데이터 시각화**: 추출된 데이터의 평균 처리 시간을 계산하고, 이를 그래프로 시각화하여 성능 분석에 활용합니다.

### 파일 설명

1. **`extract_data_from_log.py`**:
    - **역할**: 로그 파일에서 데이터를 추출하고, 이를 Elasticsearch에 저장하며, 데이터의 시각화를 담당합니다.
    - **주요 기능**:
        - `watchdog` 라이브러리를 사용하여 로그 파일의 변화를 실시간으로 감지하고, 새로운 로그 파일이 생성되거나 이동될 때마다 자동으로 데이터를 처리합니다.
        - 로그 파일에서 요청 및 응답 데이터를 추출합니다.
        - 추출된 데이터를 Elasticsearch에 저장합니다.
        - 데이터의 평균 처리 시간을 계산하고, 이를 그래프로 시각화합니다.
        - 로그 파일의 변경을 감지하여 자동으로 데이터를 처리합니다.
    - **주의사항**
        - 새로운 형식의 로그를 추가할 때는, 본 py파일에 로그 파싱룰을 추가해주어야 합니다.

2. **`log_files`**:
   - **역할**: 로그 파일이 저장되는 디렉토리입니다.
   - **주요 기능**: 로그 파일을 저장하고, `extract_data_from_log.py`에서 이 파일들을 읽어들여 데이터를 처리합니다.

3. **`log_backup`**:
   - **역할**: 오픈 후, 로그가 유실되는 경우를 방지하기 위해, 로그 파일을 저장하는 디렉토리입니다.
   - **기타사항**: 안정화되면 이 폴더를 삭제하고, `extract_data_from_log.py`에서 이 폴더에 저장하는 코드도 삭제해줍니다.
